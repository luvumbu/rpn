Aperçu

RPN est une plateforme communautaire : les membres se connectent via Google, les administrateurs via les identifiants MySQL. Le code suit une architecture MVC légère, sans framework, avec un autoloader maison.

  • Connexion membre : Google OAuth 2.0
  • Connexion admin : identifiants MySQL réels (vérifiés par connexion)
  • Gestion des membres, rôles, blocages d'IP, paramètres, thèmes

Bootstrap & routage

Chaque point d'entrée charge bootstrap.php qui :

  • charge config/config.php et helpers.php
  • enregistre un autoload (cherche dans core/, controllers/, models/)
  • démarre la session

Le routage est simple : un fichier = une route. admin.php utilise un mini-routeur :

$action = $_GET['action'] ?? 'login';
switch ($action) {
    case 'dashboard': $admin->dashboard(); break;
    case 'members':   $admin->members();   break;
    case 'security':  $admin->security();  break;
    case 'promote':   $admin->promote();   break;
    // ...
}

Flux OAuth Google

  1. index.php génère un state (anti-CSRF) et le lien Google.
  2. L'utilisateur s'authentifie chez Google et accepte.
  3. Google redirige vers google-callback.php avec un code.
  4. GoogleClient::fetchToken() échange le code contre un access_token.
  5. GoogleClient::fetchUserInfo() récupère nom / email / photo.
  6. User::upsertFromGoogle() enregistre/maj le membre en base.
  7. L'utilisateur est mis en session → redirection vers dashboard.php.
L'URI de redirection est calculée automatiquement (schéma + hôte) → fonctionne en local et en ligne sans modification.

Base de données

Une seule table users :

id          INT PK AI
username    VARCHAR(50)   -- identifiant admin éventuel
google_id   VARCHAR(64)
name        VARCHAR(150)
email       VARCHAR(190)  UNIQUE
picture     TEXT
role        ENUM('membre','admin') DEFAULT 'membre'
password    VARCHAR(255)  -- (réservé, non utilisé par l'admin MySQL)
blocked     TINYINT(1)    DEFAULT 0
created_at  TIMESTAMP
last_login  TIMESTAMP

Création automatique : à la première connexion, si la base n'existe pas elle est créée (en local), et la table est garantie via CREATE TABLE IF NOT EXISTS.

Classes core

Database

  • pdo() — connexion PDO partagée (singleton), auto-création de la base/table.
  • tryConnect($host,$name,$user,$pass) — teste des identifiants (login admin).
  • persist($db) — enregistre les identifiants (storage/ ou config/) pour tout le site.
  • Identifiants effectifs : session (admin) en priorité, sinon config.

GoogleClient

  • getAuthUrl($state), fetchToken($code), fetchUserInfo($token)
  • cURL centralisé dans request() (une seule fonction réutilisée)
  • Clés lues via Settings (modifiables dans l'admin)

LoginGuard

  • Anti-bruteforce basé sur un fichier (storage/login_attempts.json)
  • recordFailure(), isBlocked(), attemptsLeft(), reset(), resetAll(), blockedList()
  • Seuil & durée lus via Settings

Settings

  • get($cle,$defaut) / save($valeurs) → storage/settings.json
  • Stocke : clés Google, seuil/durée de blocage, thème, message principal

Theme

  • all() (4 thèmes), key() (thème actif), css() (variables CSS)
  • Injecté dans le <head> de chaque vue

Session & helpers

  • Session : set/get/has/user/isAdmin/destroy
  • helpers.php : redirect($url), view($nom,$data)

Espace admin

Connexion via identifiants MySQL : tryConnect() tente une vraie connexion. Si elle réussit → accès accordé. Les identifiants sont gardés en session (et persistés en ligne pour les membres).

En local, l'utilisateur MySQL est root ; en ligne (Hostinger), l'utilisateur = le nom de la base.

Hiérarchie

ActionSuper-adminAdmin normal

Super-admin = connexion MySQL (id 0) ou email dans ADMIN_EMAILS. Protections appliquées côté serveur.

Sécurité

  • Anti-bruteforce : N essais → blocage IP H heures (réglable). Bouton désactivé si bloqué.
  • CSRF : paramètre state vérifié au retour Google.
  • PDO préparé : protection contre l'injection SQL.
  • Dossiers protégés : config/, storage/, app/ via .htaccess.
  • URL secrète : unlock.php?key=… (faux 404 sans la clé).
Ne jamais publier SECRET_KEY, config/db.php ni le Client Secret Google.

Thèmes

4 thèmes : Panafricain, Classique, Sombre, Clair. Chaque thème définit des variables CSS injectées par Theme::css(). Les vues utilisent ces variables (--bg-base, --text, --accent, --bar…).

Aperçu en direct : sur la page Paramètres, un script applique les variables du thème sélectionné sans recharger.

select.addEventListener('change', () => {
    const vars = THEMES[select.value];
    for (const k in vars) document.documentElement.style.setProperty(k, vars[k]);
});

Configuration (config/config.php)

GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET   // clés OAuth (surchargées par l'admin)
GOOGLE_REDIRECT_URI                       // calculée automatiquement
DB_HOST / DB_NAME / DB_USER / DB_PASS     // local par défaut, ou config/db.php
ADMIN_EMAILS                              // super-admins (Google)
SECRET_KEY                                // clé de la page unlock.php
IS_LOCAL                                  // détection local / en ligne
En ligne, les identifiants BDD viennent de config/db.php (créé manuellement ou à la 1re connexion admin).
