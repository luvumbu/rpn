⚙️ Comment ça marche

Une PWA, c'est un site web normal + un manifeste + un service worker. Le navigateur (Chrome, Edge, Safari…) détecte ces fichiers et propose alors d'« installer » le site comme une application : icône sur l'écran d'accueil, fenêtre dédiée, plein écran.

Pour l'APK Android, on n'écrit pas d'application Android : on utilise PWABuilder, qui crée une TWA (Trusted Web Activity) — une mini-coquille Android qui ouvre le site RPN en plein écran. L'APK « enveloppe » donc le site en ligne.

⚠️ La TWA charge le site en ligne : l'APK a besoin que RPN soit déployé sur une URL HTTPS publique (ex. https://bokonzi.com/rpm/). En local (http://localhost) la PWA s'installe, mais l'APK final pointera vers l'URL publique.

📲 Installer l'application (PWA)

Directement depuis le navigateur, sans store :

AppareilProcédure
Android (Chrome)Ouvrir le site → menu ⋮ → « Installer l'application » / « Ajouter à l'écran d'accueil ».
iPhone / iPad (Safari)Ouvrir le site → bouton Partager → « Sur l'écran d'accueil ».
Windows (Edge/Chrome)Icône d'installation dans la barre d'adresse, ou menu → Applications → « Installer ce site ».
macOS (Chrome/Safari)Idem Windows (barre d'adresse ou menu).

Une fois installée, l'app apparaît avec l'icône RPN et s'ouvre en plein écran, comme une vraie application.

🤖 Générer l'APK / AAB (PWABuilder)

Exactement la même méthode que pour le Blog :

  1. Déployer RPN en ligne en HTTPS (ex. https://bokonzi.com/rpm/) et vérifier que https://bokonzi.com/rpm/manifest.json s'ouvre bien.
  2. Aller sur https://www.pwabuilder.com, coller l'URL https://bokonzi.com/rpm/ et lancer l'analyse (le score doit valider le manifeste, le service worker et les icônes).
  3. Cliquer Package For Stores → Android. Régler le Package ID (ex. com.bokonzi.rpm), le nom de l'app et la version, puis Generate.
  4. Télécharger le .zip. Il contient : app-release-signed.apk (test direct), app-release-bundle.aab (pour le Play Store), signing.keystore + signing-key-info.txt, et assetlinks.json.
  5. Installer le .apk sur un téléphone Android pour tester (autoriser « sources inconnues »).
🔐 Conserve précieusement signing.keystore et son mot de passe (signing-key-info.txt) : ils sont indispensables pour publier les futures mises à jour sur le Play Store. Perdu = impossible de mettre à jour l'app publiée.

💡 Range ces fichiers dans un dossier APK/ du projet (comme blog/APK/), hors dépôt public.

🔗 assetlinks.json (supprimer la barre d'adresse)

Pour que la TWA s'ouvre en plein écran (sans la barre d'URL du navigateur), Android doit vérifier que l'app et le site t'appartiennent. PWABuilder fournit un fichier assetlinks.json à publier sur le serveur :

https://bokonzi.com/.well-known/assetlinks.json

Il contient l'empreinte SHA-256 de ta clé de signature, par exemple :

[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "com.bokonzi.rpm",
    "sha256_cert_fingerprints": ["AB:CD:..."]
  }
}]
Le dossier .well-known doit être à la racine du domaine (bokonzi.com/.well-known/), pas dans /rpm/.

🏪 Publier sur le Play Store (optionnel)

  1. Créer un compte développeur Google Play (frais unique de 25 $).
  2. Nouvelle application → téléverser le .aab.
  3. Remplir la fiche (description, captures d'écran, icône 512, politique de confidentialité).
  4. Soumettre à validation. Mêmes package_name et keystore à réutiliser pour chaque mise à jour.

Tu peux aussi simplement distribuer l'APK (lien de téléchargement) sans passer par le Store.

🎨 Changer l'icône de l'app

Modifie la fonction New-RpmIcon dans generate-app-icons.ps1, puis régénère :

powershell -ExecutionPolicy Bypass -File .\generate-app-icons.ps1

Cela réécrit icon-192.png et icon-512.png. Incrémente ensuite le ?v= des icônes dans manifest.json et Theme::pwa() pour forcer le rafraîchissement.

Le système met l'icône d'une PWA installée en cache : pour la voir changer, désinstalle/réinstalle l'app (ou attends que le navigateur redétecte le manifeste).

🔄 Mettre à jour l'application

  • PWA installée : rien à faire — comme le site charge en ligne, toute modif du code PHP est visible immédiatement à la prochaine ouverture (le service worker ne cache pas les pages).
  • APK déjà publié : régénérer un .aab via PWABuilder avec une version supérieure, signé avec le même keystore, puis le téléverser sur le Play Store. (Le contenu, lui, se met à jour tout seul puisque la TWA charge le site en ligne — un nouvel APK n'est nécessaire que pour changer l'icône, le nom ou les permissions.)
