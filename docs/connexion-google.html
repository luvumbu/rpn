🔑 C'est quoi la connexion Google ?

Les membres de RPN se connectent avec leur compte Google (bouton « Se connecter avec Google »), via le protocole OAuth 2.0. Aucun mot de passe n'est stocké : Google authentifie l'utilisateur et renvoie son profil (nom, e-mail, photo), que RPN enregistre en base.

Pour que Google accepte de rediriger vers ton site, il faut déclarer ton site dans la Google Cloud Console (l'URL exacte de retour, dite « URI de redirection »).

La connexion administrateur (identifiants MySQL) n'utilise PAS Google : elle fonctionne indépendamment, même si Google n'est pas configuré.

⚙️ Comment ça marche (le flux)

  1. L'utilisateur clique « Se connecter avec Google » sur l'accueil /rpn/.
  2. Il est envoyé sur Google (avec un paramètre state anti-CSRF).
  3. Après accord, Google le renvoie vers l'URI de redirection : …/rpn/google-callback.php.
  4. RPN échange le code reçu contre un token, récupère le profil (nom, e-mail, photo).
  5. Le membre est créé/mis à jour en base, puis redirigé vers son tableau de bord.

🌐 La page Google (où configurer)

Tout se passe dans la Google Cloud Console, section Identifiants :

https://console.cloud.google.com/apis/credentials
  1. Connecte-toi avec le compte Google propriétaire de l'app (fjs.troubadourstory@gmail.com).
  2. Vérifie le bon projet sélectionné en haut (menu déroulant à côté de « Google Cloud »).
  3. Va dans API et services → Identifiants (ou le lien ci-dessus).
  4. Sous « ID clients OAuth 2.0 », clique sur le tien (type Application Web).
  5. Modifie les URI de redirection et origines JavaScript (section suivante), puis Enregistre.
Si le lien direct n'ouvre pas la bonne page : console.cloud.google.com → menu ☰ → API et services → Identifiants.

Pendant la transition, tu peux garder aussi les anciennes URL en /rpm/ : les deux versions fonctionneront en parallèle.
⚠️ L'URI de redirection doit être strictement identique à celle qu'envoie le site (protocole, domaine, casse, chemin). La moindre différence ⇒ erreur redirect_uri_mismatch.

🗝️ Les clés dans le projet

Dans config/config.php :

define('GOOGLE_CLIENT_ID',     '…apps.googleusercontent.com');
define('GOOGLE_CLIENT_SECRET', 'VOTRE_SECRET_GOOGLE');
// URI de redirection : construite AUTOMATIQUEMENT (http/https + domaine)
define('GOOGLE_REDIRECT_URI', $__scheme . '://' . $__host . '/rpn/google-callback.php');
  • Client ID / Secret : fournis par Google Cloud Console (à ne jamais publier).
  • Redirect URI : le code détecte tout seul http/https et le domaine ; seul le chemin /rpn/google-callback.php est fixe.
  • Les clés sont aussi modifiables depuis Admin → Paramètres (elles surchargent le fichier).
⚠️ Ne publie jamais config/config.php (il contient le CLIENT_SECRET).

🔁 Changement de dossier : rpm → rpn

L'application a été déplacée du dossier rpm vers rpn. La connexion Google dépend du chemin de l'URI de redirection : il faut donc ajouter les nouvelles URL /rpn/ côté Google (section « URL à mettre »).

ÉlémentAvantMaintenant
Dossier / URL…/rpm/…/rpn/
URI de redirection…/rpm/google-callback.php…/rpn/google-callback.php

✅ La connexion admin (MySQL) marche déjà sur /rpn/ sans rien configurer. Seule la connexion Google nécessite d'ajouter les URI /rpn/.

🛡️ Sécurité

  • Anti-CSRF : un paramètre state aléatoire est généré puis vérifié au retour.
  • Secret protégé : le CLIENT_SECRET reste côté serveur, jamais exposé au navigateur, jamais publié.
  • Pas de mot de passe stocké : RPN ne conserve que le profil renvoyé par Google.
  • Admins automatiques : les e-mails listés dans ADMIN_EMAILS (config) deviennent admin à la connexion.
