1. Présentation

RPN est une plateforme communautaire : articles, agenda de rendez-vous entre membres, notifications, et une application installable sur téléphone.

RPN réunit en un seul site :

  • 📰 un magazine d'articles (lecture, écriture, mise en page, avis, discussion) ;
  • 📅 un agenda d'événements entre membres (créneaux, réservations, photos, notes, carte de proximité) ;
  • 🔔 des notifications en temps réel (comme des SMS) ;
  • 🛡️ un espace administrateur complet ;
  • 📱 une application installable (PWA / APK) avec mises à jour.
Le site s'adapte au thème choisi (couleurs) et fonctionne sur ordinateur comme sur téléphone.

3. Se connecter

Trois portes : Google, e-mail/mot de passe (membres) et identifiants (administrateur).

👤 Membre — « Se connecter avec Google »

  1. Sur la page d'accueil, clique « Se connecter avec Google ».
  2. Choisis ton compte Google et autorise.
  3. Tu arrives sur ton tableau de bord. Ton profil (nom, photo) est repris de Google.

✉️ Membre — sans compte Google (inscription par e-mail)

Pas de compte Google ? Tu peux créer un compte classique :

  1. Sur l'accueil, sous le bouton Google, utilise le formulaire e-mail + mot de passe pour te connecter, ou clique le bouton vert « S'inscrire ».
  2. Renseigne nom, e-mail, mot de passe (min. 6 caractères, à confirmer).
  3. Tu es connecté·e directement et tu peux ajouter ta photo depuis le tableau de bord (voir §4).
Pour ta sécurité, l'inscription par e-mail crée toujours un compte membre (jamais administrateur). Les mots de passe sont chiffrés.

🛡️ Administrateur

Va sur …/rpn/admin/login et saisis le nom de la base et le mot de passe MySQL. C'est aussi la page de configuration du site (voir la doc « Connexion Google » et l'installation).

La configuration détaillée de la connexion Google (URI de redirection, console) est expliquée dans connexion-google.html.

Ta page d'accueil une fois connecté·e : …/rpn/dashboard.

On y trouve :

  • Profil : avatar, nom, badge (Membre / Administrateur), e-mail. Tu peux 📷 ajouter / changer / retirer ta photo ici (utile surtout si tu t'es inscrit·e par e-mail). Un bouton 🔄 Mise à jour (juste sous l'e-mail) vérifie la dernière version de l'app.
  • 🏅 Mon niveau : ta progression Héritier → Sage, avec un lien « Voir tous les niveaux » (voir §9e).
  • 📌 Tes rendez-vous aujourd'hui : rappel des événements réservés du jour (heure, titre, hôte, lieu), cliquables vers l'agenda.
  • 🪪 Mon code & visibilité : ton code membre et l'autorisation d'être ajouté (voir §5).
  • 🌍 Mes pays d'origine et 🎓 Mon rôle / domaine + 📍 adresse : ton profil, avec autocomplétion (voir §9d).
  • 🔎 Rechercher un membre : barre de recherche avec autocomplétion des profils visibles (voir §9f).
  • 🎨 Mon thème : choisis ton apparence perso (voir §6).
  • Statistiques : articles publiés, tes articles, questionnaires (cliquables).
  • Actions rapides, rangées par thème : 👥 Communauté (Rechercher un profil, Messages, Agenda, Carte diaspora), 📚 Apprendre & contenus (Articles, Questionnaires, Assistant), 🧭 Mon espace (Niveaux, Notifications, Export/Import, Aide), et 👑 Administration si tu es admin.
  • 📦 Sauvegarde de mon projet : tout en bas, pour exporter/importer tes contenus en .zip.
Besoin d'aide en direct ? Le bouton flottant 🎓 Visite guidée (en bas à gauche, sur toutes les pages) surligne chaque élément et l'explique, page après page.

5. Code membre & visibilité

Chaque membre a un code unique et décide lui-même s'il peut être ajouté à des activités. C'est le cœur du respect de la vie privée sur RPN.

🪪 Ton code — à quoi ça sert ?

Chaque membre reçoit un code unique de la forme 1 lettre + 3 chiffres (ex. G235), affiché sur son tableau de bord. Aucun code ne se répète.

Ce code est une clé d'accord privée. Concrètement, il sert à :

  • Être ajouté à un événement sans être public : tu donnes ton code à un organisateur de confiance ; lui seul peut alors t'inscrire à son créneau, même si tu restes invisible pour tous les autres.
  • Te retrouver directement : quelqu'un qui connaît ton code peut t'ajouter sans avoir à fouiller dans une liste.

👉 En résumé : connaître ton code = tu as donné ton accord. Tant que tu ne le partages pas, personne ne peut t'inscrire par ce moyen.

🔓 « Trouvable » ou « invisible » ?

Par défaut, tu es invisible : tu n'apparais dans aucune liste publique, et personne ne peut t'ajouter à une activité (sauf si tu as donné ton code, voir ci-dessus). C'est le mode « vie privée ».

Si tu cliques sur « M'autoriser à être ajouté », tu deviens trouvable : ton nom apparaît dans la liste que voient les organisateurs, et ils peuvent te cocher pour t'inscrire — sans avoir besoin de ton code. Tu peux redevenir invisible à tout moment d'un seul clic.

Méthode pour t'ajouter à un événementCondition
Depuis la liste publique (l'organisateur coche ton nom)Il faut avoir activé « trouvable »
Par code (l'organisateur saisit ton code)Marche toujours — c'est toi qui as transmis le code
Le code fonctionne toujours, même quand tu es invisible : partage-le seulement aux personnes de confiance. L'option « trouvable » est un raccourci public en plus — pas une obligation.
Aucune de ces options ne révèle ton e-mail ni ne te notifie sans raison : un ajout déclenche simplement une notification de confirmation.

6. Mon thème (personnel)

Change l'apparence du site rien que pour toi.

  1. Sur le tableau de bord, carte « 🎨 Mon thème ».
  2. Choisis un thème dans la liste — l'aperçu est immédiat.
  3. Clique « Appliquer ». Ton choix est mémorisé et te suit à chaque connexion.

Les thèmes sont rangés par famille dans le menu :

  • Communauté : Panafricain (défaut), Pro Black.
  • Clairs : Classique, Clair, Menthe.
  • Sombres & néon : Sombre, Océan, Forêt, Coucher de soleil, Violet.
  • Héros & dessins animés : Simpson, Black Panther, Batman, Rayman.
  • Sur mesure : Personnalisé (couleurs définies par l'admin).

« Thème du site (défaut) » remet l'apparence commune.

Ton thème ne s'applique qu'à toi. Les autres gardent le thème du site (défini par l'admin, mêmes familles dans Paramètres).

7. Lire les articles

Page …/rpn/articles — accessible aux visiteurs.

  • Liste des articles publiés, avec recherche (filtre par titre/contenu).
  • Clique un article pour l'ouvrir : sa mise en page dépend du modèle choisi (image en haut, carrousel, etc.).
  • Compteur de vues : une vue par visiteur (par IP) et le suivi des membres.
  • Galeries animées : selon le modèle, les photos défilent (carrousel à flèches, diaporama en fondu, galerie à défilement continu).
📣 Les articles passés en « Annonce » par un admin apparaissent en avant sur la page d'accueil de tout le monde (voir §24).

8. Écrire / modifier un article

Tout membre peut écrire. Bouton « ＋ Nouvel article » dans la liste.

  1. Titre et contenu (texte enrichi).
  2. Image de couverture (facultative) + galerie de photos.
  3. Fichiers joints (PDF, documents…) téléchargeables par les lecteurs.
  4. Mise en page : choisis parmi ~19 modèles. Aperçu en direct.
  5. Article parent (facultatif) : rattache l'article à un autre pour en faire un sous-article (il s'ouvre alors depuis son parent, et apparaît en petite vignette sur la carte du parent).
  6. 📝 Questionnaires associés (facultatif) : coche un ou plusieurs questionnaires (voir §9c). Ils seront proposés à la fin de la lecture de l'article.
  7. 🟥 Afficher dans le tableau de bord de tout le monde : coche pour signaler l'article comme URGENT (alerte rouge chez tous les membres).
  8. 🔑 Protéger par mot de passe (facultatif) : 3 choix — Aucun (libre), Mon code membre (ton code sert de mot de passe par défaut, à communiquer aux lecteurs), ou Personnalisé (tu saisis ton propre mot de passe). Voir §8b.
  9. Brouillon ou Publié : un brouillon n'est visible que de toi (et des admins). Au clic, si le titre ou le contenu manque, un message te le signale (le champ concerné se surligne).
Sur la page d'un article, la photo de l'auteur s'affiche en rond en haut à droite. L'auteur (et les admins) voient aussi qui a lu l'article : la liste des membres + les adresses IP des visiteurs non connectés (voir §24).

8b. Accès protégé par mot de passe

Un article peut exiger un mot de passe pour être ouvert.

  • Côté auteur : dans le formulaire, bloc « 🔑 Protéger l'accès » — choisis Aucun, code membre (ton code, ex. G235) ou mot de passe personnalisé.
  • Côté lecteur : à l'ouverture, un écran demande le mot de passe. Une fois saisi correctement, l'accès est mémorisé pour la visite.
  • L'auteur et les admins accèdent sans mot de passe.
  • Dans la liste, un article protégé affiche « 🔒 Contenu protégé » au lieu de l'extrait (pas de fuite).
Le mot de passe est chiffré et n'est jamais affiché. Pour le retirer, repasse l'option sur « Aucun ».

9. Avis & discussion

Chaque article peut être noté et commenté.

  • Avis : une note de 1 à 5 ★ + un commentaire (un avis par membre, modifiable).
  • Discussion : fil de messages sous l'article. Un message peut être signalé (modération) et supprimé par son auteur ou un admin.
  • Suppression de l'article : par son auteur ou un administrateur.

9b. Signaler un article

Un système de modération communautaire pour les contenus inappropriés.

  • Sur la page d'un article, tout membre connecté (sauf l'auteur) voit un bouton 🚩 Signaler — un seul signalement par personne.
  • Dès que 3 membres différents ont signalé un article, il est automatiquement masqué au public.
  • Conditions : un article protégé ou en annonce n'est jamais masqué ; l'auteur et les admins continuent de le voir, avec un bandeau d'avertissement.
  • Un administrateur peut réinitialiser les signalements (ce qui rétablit l'article) — directement depuis l'article, ou depuis Admin → Articles (panneau « 🚩 Articles signalés » qui liste tout d'un coup).

9c. Questionnaires interactifs (QCM)

Crée des quiz notés, associe-les à des articles, et propose-les en fin de lecture. Accès : tableau de bord → « ❓ Questionnaires » (ou Admin → Questionnaires).

✍️ Créer un questionnaire

  1. Clique « ➕ Créer un questionnaire ». Donne un titre, une description et, si tu veux, une image de couverture.
  2. Ajoute autant de questions que tu veux. Pour chaque question, choisis le mode :
    • 🔘 Une seule bonne réponse (choix unique) ;
    • ☑️ Plusieurs bonnes réponses (choix multiple).
  3. Saisis les réponses possibles et coche la (ou les) bonne(s).
  4. Publie (ou garde en brouillon). N'importe quel membre peut créer un quiz ; l'auteur et les admins peuvent le modifier/supprimer.

🎯 Répondre, score & pourcentage

Un membre répond, valide, puis voit son score (ex. 7/9), son pourcentage de réussite (avec une barre de progression) et le corrigé (bonne réponse en vert, son choix faux en rouge, réponse manquée signalée). Le pourcentage apparaît aussi sur la carte du quiz et sous l'article associé.

🔁 Nombre de tentatives

À la création, le créateur fixe le nombre de tentatives autorisées :

  • 0 = illimité : le membre peut refaire le quiz autant de fois qu'il veut.
  • 1, 2, 3… : au-delà, le bouton « Refaire » se verrouille (« Nombre maximum de tentatives atteint »).

La limite est ignorée pour les questionnaires obligatoires (pour ne jamais bloquer un membre qui doit le valider).

🔗 Associer à un article

Dans le formulaire d'article, section « 📝 Questionnaires associés », coche les quiz à rattacher. En bas de l'article, un encadré « Teste tes connaissances » proposera « Commencer à répondre → » (avec l'image et le score si déjà fait).

⛔ Questionnaire obligatoire (admin)

Un administrateur peut cocher, à la création :

  • « Rendre obligatoire » : tant qu'un membre n'a pas répondu, toute l'application est bloquée pour lui (il est redirigé vers le questionnaire). Les admins ne sont jamais bloqués.
  • « Il faut réussir pour continuer » : le membre doit obtenir toutes les bonnes réponses pour être débloqué (sinon il recommence). Sans cette option, répondre suffit.
Comme les articles, un quiz peut aussi être marqué 🟥 URGENT pour apparaître dans le tableau de bord de tous.

9d. Mon profil : rôle, pays d'origine & adresse

Depuis le tableau de bord, complète ton profil pour être trouvé·e par la communauté. Trois blocs distincts :

  • 🌍 Mes pays d'origine : un champ avec autocomplétion — tape « séné » → choisis 🇸🇳 Sénégal (drapeau affiché). Plusieurs pays possibles ; chaque pays devient une étiquette (clique « × » pour retirer).
  • 🎓 Mon rôle / domaine : un seul champ qui se complète automatiquement — tape « maths » → propose « Prof de maths », « Mathématiques »… Clique pour ajouter, ou écris ton propre rôle puis Entrée.
  • 📍 Mon adresse / ville : champ avec autocomplétion de villes. La ville choisie est géolocalisée : tu apparais sur la carte de la diaspora et dans la recherche par kilomètres.
Pour être visible des autres (recherche, carte), active « M'autoriser à être ajouté » dans « Mon code & visibilité ». Sinon ton profil reste privé.

9e. Mes niveaux (Héritier → Sage)

Tu gagnes des points en participant. Une carte sur le tableau de bord montre ton niveau, ta progression et le détail ; la page « Mes niveaux » liste tous les paliers et le barème.

  • 📰 Article publié +15 · ❓ QCM passé +10 · 📅 Réservation +5 · ⭐ Avis +3 · 💬 Message +2 · ⏳ ancienneté +2/semaine.
  • Paliers : 🌱 Héritier (0) → 🤝 Transmetteur (50) → 🛡️ Gardien (150) → 🏗️ Bâtisseur (350) → 🦉 Sage (700).

9f. Rechercher un membre / professeur

Depuis « Rechercher un profil » (Actions rapides) ou la carte « Rechercher un membre » : tape un nom, une matière, un pays ou une ville.

  • Autocomplétion dès 2 lettres : suggestions avec avatar, nom et matière — uniquement les profils visibles.
  • Les résultats affichent le niveau, les matières, le pays (drapeau) et la distance en km entre toi et la personne (les plus proches d'abord).
  • Les filtres de matières n'affichent que celles réellement portées par des membres (avec le nombre).

9g. Messagerie privée

Discute en privé avec un autre membre : depuis une fiche, clique « ✉️ Message », ou ouvre « Messages » (Actions rapides) pour voir toutes tes conversations.

  • Boîte de réception avec compteur de non-lus (badge sur le tableau de bord).
  • Temps réel adaptatif : si ton interlocuteur est en ligne (point vert), la conversation se met à jour toutes les 10 s ; sinon toutes les 60 s.

9h. Carte de la diaspora

« Carte de la diaspora » (Actions rapides) affiche une carte interactive : les grands foyers afro-descendants (Afrique, États-Unis, Canada, Caraïbes, Guyane, Brésil, Europe) et les membres trouvables géolocalisés (points dorés). Renseigne ton adresse pour y apparaître.

9i. Assistant Sankofa

L'Assistant Sankofa (Actions rapides) n'est pas une IA : c'est un algorithme qui cherche dans les articles et les questionnaires et t'oriente. Pose une question ou un mot-clé (« lingala », « réserver un cours », « prof de maths ») → il propose les bonnes ressources et les bons raccourcis.

10. Le calendrier

Page …/rpn/agenda — vue d'ensemble façon agenda.

  • Vues mois / semaine / jour, navigation avant/arrière.
  • Couleurs : mes créneaux, mes réservations, événements publics (option à activer).
  • Clique « + » sur un jour pour créer un événement à cette date.
  • Onglets : Calendrier, ＋ Créer un événement, Mes créneaux, Événements passés, recherche.

11. Créer un événement

Onglet « ＋ Créer un événement ».

  1. Titre, date et heure (obligatoirement dans le futur).
  2. Durée et nombre de places.
  3. Type : 📍 Présentiel (adresse géocodée pour la carte) ou 🔗 En ligne (lien visio).
  4. Visibilité : 🌐 Public (visible par tous) ou 🔒 Privé (accessible par code). Chaque événement reçoit un code unique, visible dans « Mes créneaux ».
  5. Délai minimum de réservation : ferme les inscriptions X heures avant le début.
  6. Participants : coche dans la liste claire (nom + prénom) des membres ayant autorisé, ou ajoute-les par code (champ dédié, séparés par un espace).
  7. Photos : ajoute une ou plusieurs images de l'événement.
  8. Adresse (avec suggestions OpenStreetMap) et description.
L'option « rendre les inscrits et leurs notes visibles » est masquée par défaut (tu peux l'activer). Un événement public prévient les autres membres par notification.

12. Gérer mes événements (« Mes créneaux »)

Tu es l'hôte : tu gardes le contrôle total avant le début.

  • Code de l'événement affiché (public ou privé) + bouton « copier ».
  • Modifier titre, date/heure, durée, description : si l'horaire change, les inscrits sont prévenus.
  • Lieu et délai de réservation : modifiables, avec historique (les anciennes valeurs restent barrées).
  • Places : augmenter / diminuer.
  • Public ⇄ Privé : bascule à tout moment (le code reste stable).
  • Ajouter un membre hors limite (liste des « trouvables » ou par code).
  • Présence : après l'événement, marque qui était présent.
  • Photos : visibles sur la fiche. Supprimer l'événement efface tout (réservations, photos, notes).
  • 🔒 Protéger l'événement : il est conservé même si un admin « efface tout l'agenda ».
  • 📄 Voir la fiche complète : chaque événement a sa page dédiée (détail public — voir §13).

13. Réserver une place

Sur un événement qui n'est pas le tien.

  1. Trouve l'événement (liste publique, recherche, calendrier ou par code).
  2. Clique « Réserver une place » (si des places sont libres et le délai non dépassé).
  3. Tu reçois une confirmation ; l'hôte est notifié. Tu peux annuler à tout moment.

🔒 Événement privé par code

Dans la recherche de l'agenda, saisis le code (2 lettres + 3 chiffres) communiqué par l'hôte pour accéder à un événement privé.

📄 Fiche dédiée de l'événement

Chaque événement a une page complète (comme un article) : …/rpn/agenda/event?id=… — titre, date + heure exacte (de … à …) en plus du compte à rebours, hôte (et sa note), description, photos, carte (si présentiel), places avec % d'occupation, note de l'événement et bouton de réservation. Elle n'affiche que les événements publics (les privés restent réservés au créateur, aux inscrits et aux admins). Lien « 📄 Voir la fiche complète » sur chaque créneau.

📅 Ajouter à Google Agenda

Sur les cartes de créneaux (Mes créneaux, Mes réservations, Trouver) et sur la fiche de l'événement, un bouton bleu « 📅 Ajouter à Google Agenda » ouvre Google Agenda pré-rempli (titre, date/heure en fuseau de Paris, lieu) : un clic + confirmation et l'événement est dans ton agenda. Un lien « .ics » permet aussi de l'ajouter à tout autre agenda (Apple, Outlook…). Ce n'est pas une synchro automatique : c'est un ajout en 1 clic, sans réglage.

🔎 Rechercher un événement

Sous « Tous les créneaux publics », un champ de recherche filtre instantanément par titre, hôte ou lieu.

14. Recherche par proximité (carte)

Trouve les événements présentiels autour d'une ville.

  • Saisis une ville/adresse (ta ville par défaut est pré-remplie si tu l'as enregistrée).
  • Choisis un rayon (km) ou « Toutes » (toutes distances).
  • Filtre mes créneaux et/ou ceux des autres (publics). Les privés ne sont visibles que de leur créateur et des admins.
  • La carte (Leaflet/OpenStreetMap) affiche tous les points ; les événements au même endroit sont regroupés (clique le marqueur pour la liste).

15. Noter un événement

Les participants évaluent l'événement.

  1. Tu dois avoir été inscrit·e à l'événement.
  2. Une fois l'événement commencé, des étoiles ★ apparaissent sur sa fiche.
  3. Clique le nombre d'étoiles (1 à 5). La moyenne et le nombre d'avis s'affichent ensuite sur l'événement.
À distinguer de la réputation de l'hôte (note de la personne), qui est séparée.

16. Événements passés

Onglet « 🕒 Événements passés ».

Les créneaux terminés y sont déplacés en lecture seule : on ne peut plus les modifier, mais on garde l'historique (participants, présence, notes).

17. Notifications

Page …/rpn/notifications + alertes en temps réel.

  • Reçues quand : un événement public est créé, tu es ajouté·e à un créneau, ton créneau change d'horaire, un article est publié, etc.
  • Temps réel : l'app sonde le serveur régulièrement → les alertes arrivent quasi instantanément.
  • Comme un SMS : notification système (bannière), son et vibration sur mobile.
  • Pastille de non-lues sur l'icône 🔔. La page liste tout et marque comme lu ; tu peux vider la liste.

17b. Alertes URGENT (carré rouge)

Mettre quelque chose d'important en évidence pour tout le monde.

🟥 Marquer urgent

  • Article : dans le formulaire (admin), interrupteur rouge « 🟥 URGENT » (ou bouton dans la modération).
  • Événement : dans « Mes créneaux », bouton « 🟥 Marquer URGENT » (l'hôte).

📍 Effet

  • Une alerte rouge s'affiche en haut du tableau de bord de tous les utilisateurs, cliquable vers l'élément.
  • Un badge « 🟥 URGENT » apparaît aussi sur la carte/fiche de l'événement et l'article.
  • Chaque utilisateur ferme l'alerte avec la croix ✕ — elle disparaît seulement pour lui (les autres la voient encore).
  • Les événements urgents ne s'affichent que tant qu'ils sont à venir ; les articles tant qu'ils sont publiés. Re-clic = retirer l'urgence.

18. Pays d'Afrique

24. Modération des articles

…/rpn/admin/articles.

  • Vue d'ensemble de tous les articles (publiés + brouillons), avec Voir / Modifier / Supprimer.
  • 🚩 Articles signalés : un panneau en haut liste les articles signalés (nombre, masqué ou non) avec un bouton Réinitialiser ; un badge apparaît aussi dans le tableau (voir §9b).
  • Réattribuer un article à un autre membre (colonne « Auteur »).
  • 📦 Exporter tout en .zip : tout le projet — articles (texte, couverture, galeries, pièces jointes) et questionnaires (questions, réponses, images) avec leurs associations.
  • Importer un .zip : articles et questionnaires reviennent en brouillon, attribués à l'admin (les associations article↔quiz sont rétablies).
  • 🔒 Protéger un article : il est conservé lors d'un « effacer tous les articles ».
  • 📣 Annonce : passe l'article en annonce → il apparaît en avant sur la page d'accueil de tout le monde.
  • 🎨 Style des articles : taille du texte, police… appliqués globalement.

👁️ Qui a vu l'article ?

Sur la page d'un article, l'auteur et les admins voient une section dédiée : la liste des membres qui l'ont consulté (avec photo et date) et la liste des adresses IP des visiteurs non connectés. Le total en haut compte tous les visiteurs uniques (par IP).

25. Sécurité & zone de danger

Protection des connexions et effacement protégé.

🔒 Anti-bruteforce

Page …/rpn/admin/security : après trop d'essais ratés, une IP est bloquée un certain temps (réglable). Tu peux débloquer une IP (ou toutes). Une page secrète …/rpn/unlock existe aussi.

⚠️ Tout effacer (super-admin)

Dans le tableau de bord admin, « Effacer l'agenda » et « Effacer les articles » exigent le mot de passe de la base de données (vraie ré-authentification MySQL). Mauvais mot de passe → rien n'est supprimé. Les éléments protégés 🔒 (articles et événements) sont toujours conservés.

Ces actions sont irréversibles. L'export .zip sert de sauvegarde avant tout effacement.

26. API distante (écrire des articles)

Créer un article à distance avec la clé API.

Points d'entrée :

  • GET …/rpn/api/ping — test (renvoie un JSON).
  • POST …/rpn/api/article — crée un article. En-tête X-API-Key: <ta clé>.

Champs : title, content (obligatoires), template, active (1 publié / 0 brouillon), author_name, cover (fichier) ou image_url, galerie via photos[] ou gallery_urls.

curl -X POST "https://bokonzi.com/rpn/api/article" \
  -H "X-API-Key: VOTRE_CLE" \
  -F "title=Mon titre" \
  -F "content=<contenu HTML>" \
  -F "active=1" \
  -F "cover=@photo.jpg"
Sans clé valide → erreur 401. La clé se gère dans Paramètres → Clé API (§23).

27. Sécurité & vie privée

  • Connexion Google : aucun mot de passe stocké ; protection anti-CSRF (paramètre state).
  • Membres invisibles par défaut : on ne peut t'ajouter qu'avec ton accord (liste) ou ton code.
  • Événements privés : visibles seulement par leur créateur et les admins ; accès par code.
  • Secrets protégés : config/config.php et la base ne sont jamais accessibles par le navigateur (erreur 403).
  • Effacements sensibles protégés par mot de passe MySQL.
  • Codes uniques (membres et événements) : jamais de doublon.

28. FAQ / dépannage

❓ « Erreur de connexion à la base » / page d'installation

La base n'est pas configurée : va sur …/rpn/admin/login et saisis le nom de base + mot de passe MySQL. Tant que ce n'est pas fait, le site propose la page d'installation partout (sans boucle).

❓ redirect_uri_mismatch à la connexion Google

L'URI de redirection n'est pas déclarée chez Google. Voir connexion-google.html.

❓ Je ne vois pas un membre pour l'ajouter à un événement

Il n'a pas activé « trouvable ». Demande-lui son code et ajoute-le par code.

❓ Les drapeaux / la carte ne s'affichent pas

Ces éléments utilisent des services externes (flagcdn, OpenStreetMap) : une connexion Internet est nécessaire.

❓ Je ne reçois pas les notifications système

Autorise les notifications du site/app dans ton navigateur ou ton téléphone.
